# 人族 Command Card 与 HUD 数据接入

本轮目标是让可玩人族的命令面板和信息面板由 Warcraft III 原始对象数据驱动，并让按钮状态与运行时业务状态一致。

## 数据边界

- 单位/建筑按钮位置：`HumanUnitFunc.Buttonpos`，使用 `slot = y * 4 + x` 映射到 4×3 Command Card。
- 单位/建筑快捷键：`HumanUnitStrings.Hotkey`。
- 技能按钮位置：技能对象 `Profile.Buttonpos`；切换形态优先使用 `UnButtonpos`。
- 技能学习图标：技能对象 `Profile.ResearchArt`，不再把被动技能的 `PASBTN` 图标误当成学习图标。
- 科技按钮位置、快捷键和分级图标：升级对象的 `Buttonpos`、`Hotkey` 与逐级 `Art`。
- 建造、训练、研究和升级关系：单位对象的 `Builds`、`Trains`、`Researches` 与 `Upgrade`。

当前标准可玩人族目录为 17 个单位、16 个建筑、21 项科技。`--war3-human-ui-self-test` 会逐项检查这些关系、槽位冲突、英雄技能位置与物品栏资格。

## 标准命令位置

普通单位命令固定为：移动 `(0,0)`、停止 `(1,0)`、保持位置 `(2,0)`、攻击 `(0,1)`。单位技能按原始 `Buttonpos` 放置；农民的建造入口和生产建筑的集结点位于 `(3,1)`。

建造子页不再按数组顺序填充。11 个可建造人族建筑的原始位置恰好覆盖 slot 0–10，slot 11 保留返回按钮。训练单位、科技和建筑升级同样使用各自 rawcode 的原始坐标。

## 科技状态

研究按钮现在区分：

- `Ready`：前置、资源、队列和研究建筑都允许；
- `Unavailable`：缺资源、缺前置或建筑不可用；
- `Queued`：该科技已在玩家研究队列中；
- `Completed`：单级科技已完成或多级科技达到最高级。

多级科技完成一级后会切换下一等级图标和等级徽标；达到最高级后按钮禁用并显示“完成”。命令签名包含状态、徽标、图标与快捷键，因此研究完成会立即刷新样式。

此前遗漏的 `Rhan` 动物作战训练、`Rhri` 长管火枪、`Rhlh` 改进型伐木效率、`Rhse` 魔法岗哨已经接入；`Rhac` 加强型石工技术的研究建筑修正为伐木场。

## 哨塔三分支

农民建造的是 `hwtw` 哨塔，而不是完成态的 `hgtw` 防御塔。哨塔可原地升级为：

- slot 8：`hgtw` 防御塔，需要伐木场；
- slot 9：`hctw` 炮塔，需要车间；
- slot 10：`hatw` 神秘之塔，无额外建筑前置。

为支持这一点，通用建筑升级目录从“每种源建筑只有一个目标”扩展为“一个源建筑可有多个分支”。活动订单仍通过唯一 profile ID 校验，热快照、回放和状态哈希继续使用完整升级 profile。

## 英雄技能学习

英雄有未分配技能点时，普通 Command Card 的 slot 7 显示技能学习按钮。点击后进入学习子页，四个英雄技能按原始 `(0..3,2)` 排列。每个按钮显示当前/下一等级，并由技能点、英雄等级、等级间隔和最高等级共同决定状态。学习成功后扣除技能点并返回普通命令页。

已学习的主动技能、切换技能和被动技能都保留在普通页的原始位置；被动技能完整显示但不可点击。

## 信息、肖像与物品栏

- 攻击和护甲信息使用正式 Warcraft CommandButtons 图标；防御塔也从原始攻击数据展示攻击均值与攻击类型。
- 肖像血条、法力条显示“当前值 / 最大值”。
- 英雄中央信息区显示等级、当前经验、下一级经验和经验进度条。
- 英雄通过 `AInv` 获得 6 格物品栏。
- 研究 `Rhpm` 背包技能后，原始 `Upgrades` 中声明支持 `Rhpm` 的普通单位获得 2 格物品栏。
- 其他单位和所有建筑显示正式 Human inventory cover，不伪造可用物品格。

神秘藏宝室的购买与使用流程已经接入真实物品栏状态：商品购买成功后占用购买者的一格物品栏，并显示正式图标、名称、说明、主动/被动状态和剩余冷却。点击主动物品会立即使用或进入目标指针模式，被动物品不可误点。当前仍未开放丢弃和单位间转移。

底部界面布局直接使用原始 1600×512 Human console atlas 的像素坐标，再统一按 `320 / 512` 缩放。Command Card 的原始格距为 87 像素，不能按近似的 58 屏幕像素累加；物品栏使用原始两列三行边界。无物品栏资格时，`HumanUITile-InventoryCover` 以与底图完全相同的缩放和纵向原点覆盖六格，并放置在底图之上。英雄可使用 `--war3-rts-hero-capture` 生成专用布局验收截图。

## 寻路占地与选中圈

- 建筑不再使用手工估算矩形。运行时读取对象数据的 `UnitData.pathTex`；一个 Warcraft pathing cell 映射为 8 个模拟单位。
- Warcraft 的 pathing TGA 用红色表示不可行走、蓝色表示不可建造。当前模拟层只有一种矩形动态障碍，因此导入器保留所有受阻通道的完整外包络，不能再把 `12x12Simple` 兵营截成红色内圈的 8×8。兵营现在是 96×96、城镇大厅/城堡是 128×128；将来导航层支持逐格双通道 mask 后，可以分别保留红色行走 mask 与蓝色建造 mask。
- 建筑选中圈不再由矩形占地乘视觉常数推导，而是读取原始 `UnitUI.scale`。原始 32 坐标基础半径按 MDX 的 0.01 模型比例换成 Godot 世界尺寸，因此兵营使用 scale 5、城镇大厅使用 scale 6.3。
- 单位碰撞半径独立于攻击距离等世界距离换算，按原始 `collisionSize / 3` 导入，最小半径为 7。
- 选中圈的外半径直接等于运行时 `NavigationRadius`，不再用模型无关的 `radius × 2.8` 放大。
- 建筑升级前后的硬占地必须一致；哨塔三种分支以及城镇大厅的两级升级均由专项测试校验。

## 多单位编队信息卡

同类型单位仍组成 War3 子组：英雄优先，其次是有主动技能的单位，普通战斗单位排后；英雄子组按等级排序。为满足大规模 RTS 操作，项目有意取消经典版一次最多选择 12 个单位的限制，运行时选择集合现在保留所有仍存活的已选实体。当前子组的迷你头像会放大并以金色描边，负面状态单位以红框提示，每个头像都有独立血条和可选蓝条。

- `Tab` 切换到下一个子组，`Shift+Tab` 切换到上一个子组。
- 点击非当前子组头像只切换子组，肖像、状态和技能 Command Card 随之切换。
- 再次点击当前子组中的某个头像，选择会收敛到该单体。
- 信息卡使用 6×3 的紧凑头像网格，每页可显示 18 个单位；超过一页时，左侧显示数字页签和上一页/下一页箭头。页签窗口一次显示四个相邻页码并随当前页滚动，因此选择数量不会被 HUD 容量截断。
- 左侧页签只浏览选择集合，不改变当前 War3 子组，也不会重新发送选择命令。`Tab`/`Shift+Tab` 与点击头像仍负责子组切换；当当前子组改变时，HUD 会自动跳到包含该子组首个单位的页。
- 单位头像槽严格保持正方形，贴图由独立 `TextureRect` 铺满，按钮本身只负责点击和边框，因此不再出现贴图右侧的空白内容区。
- 移动、停止、保持位置等基础命令仍发送给整支编队；技能、学习技能和工人建造菜单只读取当前子组，避免混合编队里错误显示或调用别的类型的能力。
- 框选和 Shift 追加都写入完整集合；选择上限等于当前可选的存活实体数量，而不是任意 UI 常量。

规则核对来源：Blizzard 经典 Battle.net 的 [Hot Keys and Special Commands](https://classic.battle.net/war3/basics/specialcommands.shtml) 与 [Unit FAQ](https://classic.battle.net/war3/basics/unitfaq.shtml)。

经典资料用于核对 War3 的子组优先级和 `Tab` 语义；大选择集合与分页布局则采用本项目的扩展设计，并参考《星际争霸 II》的选择面板视觉语言：[Inven 的 StarCraft II 界面指南与截图](https://www.inven.co.kr/webzine/news/?news=81846&site=sc2)、[Blizzard 论坛关于 selection panel 类型选择的说明](https://us.forums.blizzard.com/en/sc2/t/shortcut-to-instantly-select-unit-type-after-cycling-them/22119)。

## 建筑建造进度

选中处于预留、等待建造者或正在建造状态的建筑时，普通信息栏显示 Human 原版 `BuildProgressBar` 边框、填充纹理和实时百分比。完成后该条自动隐藏；生产、研究或建筑升级队列仍使用独占的队列信息卡，不会与建造进度互相覆盖。

## 小地图

HUD 每次快照会把可见世界区域投影成 `CameraViewBounds`，小地图用双描边矩形显示当前镜头范围。正常模式使用普通箭头光标；只有 `MinimapSignalMode` 为真时才使用瞄准光标。当前玩法尚未开放信号模式，因此正常小地图不会再一直显示瞄准样式。

## 肖像、观察选择与阵营权限

- 单位肖像持续播放原始 `Portrait` 序列；己方单位播放选择语音时切入 `Portrait Talk`，结束后回到肖像待机。
- 建筑没有独立 portrait 模型时使用本体的 `Portrait` 序列。回退相机偏移会跟随 Warcraft 模型导入时的坐标轴修正，不再从错误侧面观察建筑。
- 没有法力值的对象仍保留不透明的深色法力槽底板，只隐藏蓝色填充与数值，避免肖像视口或世界场景从 UI 镂空处漏出。
- 可见的敌方和中立单位可以左键单选并查看生命、护甲和静态肖像；它们不生成命令卡，不播放本地选择语音，也不播放肖像动画。框选只收集己方单位，Shift 也不会把不同控制权的对象混入同一选择集合。

## 神秘藏宝室

`hvlt` 的九件商品按原始 `Makeitems`、`ItemData.slk` 和按钮坐标进入 Command Card：恢复卷轴、小净化药水、机械类的小玩艺、生命药水、魔法药水、回城卷轴、象牙塔、火焰之球和避难权杖。运行时分别维护每座商店、每件商品的最大库存与补货计时。

- 只有带有效物品栏的己方单位进入建筑边缘 150 游戏单位范围内，商店按钮才可购买；多名候选者按距离和实体 ID 稳定选择最近者。
- 生命/魔法药水、回城卷轴和象牙塔需要主城，火焰之球和避难权杖需要城堡；解锁依据已完成的本地主基地类型计算。
- 商品按钮展示黄金/木材价格、当前/最大库存、补货时间、解锁要求、购买者与不可用原因。
- 购买会原子扣除资源、减少库存并写入购买者物品栏；物品栏满、缺货、资源不足、主基地等级不足或没有近距离购买者都会禁用按钮并给出对应状态。
- 恢复卷轴按 `AIsl` 在原始 600 范围、45 秒内为非机械友军恢复 225 生命；小净化药水按 `AIpl` 在 30 秒内恢复 100 魔法。所有 Warcraft 距离都通过 `4/15` 转成当前模拟世界距离。
- 生命药水与魔法药水分别立即恢复 250 生命和 150 魔法，并使用原始 `AIhe` / `AIma` 冷却组与 20 秒冷却。
- 机械类的小玩艺生成原始 `necr` 机械兔：15 生命、100 原始移动速度、350 原始视野；它注册到召唤表现映射，因此使用正确模型和图标。
- 回城卷轴选择已完成的己方/友军主基地，保持原始 5 秒施法；施法者移动、受伤或死亡会中断，完成后传送原始 1100 范围内的友军。
- 象牙塔走正式建筑放置校验与动态硬占地，创建 `hwtw` 哨塔并以原始 4 秒 start-and-release 建造流程完成，期间信息栏和世界表现都读取 ConstructionSystem 状态。
- 火焰之球是永久被动物品：攻击增加 5 点火焰伤害、获得对空目标层，并按 `AIfb` 的原始 140 范围产生近距离溅射；现有技能伤害/攻速修正仍叠加在更新后的武器配置上。
- 避难权杖选择原始 700 范围内的友军，传送到持有者最高等级的已完成主基地；普通单位恢复 10 秒、英雄恢复 15 秒，每秒 15 生命和 1 魔法，并保持 45 秒物品冷却。
- 主动消耗品只在效果被运行时接受后移除；位置无效、超距、满生命/满魔法或冷却中会保留物品并显示具体原因。

规则核对来源：Blizzard 经典 Battle.net 的 [Arcane Vault](https://classic.battle.net/war3/human/arcanevault.shtml)、[Neutral Buildings](https://classic.battle.net/war3/neutral/buildings.shtml) 和 [Hero Item Basics](https://classic.battle.net/war3/basics/heroitembasics.shtml)。

## 基础光影

主场景使用暖色方向光、四分区方向阴影距离、较低环境补光、SSAO、Filmic 色调映射和轻量 Bloom。导出 GLB 中只有 Warcraft 材质明确设置 `Unshaded` 或使用 additive/modulate 混合的层继续保持无光照；普通不透明/透明测试层恢复逐像素受光，因此单位、建筑和树木能够接收明暗，建筑保留低成本阴影代理，静态树批次也参与投影。
