# 终点协作收敛 V2

更新日期：2026-07-10

## 1. 目标问题

V1 已经提供唯一目标槽位、Hungarian 初始分配和两单位局部换槽，但无法保证解决下面的拓扑死锁：

```text
外围槽位先被占据
    ↓
形成稳定或缓慢微调的单位环
    ↓
后续单位持有内部预留槽位
    ↓
单位存在微小进展或反复短暂 Arrived，但长期无法稳定就位
```

这类问题不能只依赖普通 Steering 或“完全零进展”检测。单位可能持续以每秒一两像素蠕动、被推入 Arrived 后又被推出，导致通用恢复不断重置。

## 2. 第一层：驻留历史与 Overflow 兜底

### 独立终点历史

每个 32 人以上移动组维护：

- `DestinationBestDistances`：当前终点阶段观察到的最佳距离。
- `DestinationStallTicks`：未改善至少 3px 的连续 Tick。
- `DestinationNearTicks`：进入目标区域后的累计驻留 Tick。
- `DestinationOverflowed`：是否已经获得 Overflow 槽位。

局部两单位换槽只重置短期停滞，不清除累计驻留历史。`WaitingForPath` 和短暂 Arrived 也不会错误清空历史。

### 触发条件

- 只处理 32 人以上编队。
- 第一次进入距离自身槽位 180px 的区域后开始跟踪。
- 已进入后使用 260px 退出滞回，避免碰撞推挤导致反复进出。
- 终点阻挡先交给主动 Yielding；只有总驻留达到 900 Tick 后才允许新 Overflow，给临时让路留下处理窗口。
- 每 30 Tick 检查一次，每次最多处理 2 个单位。

“已就位阻挡者”包括严格 Arrived，以及距离自身槽位 18px 内且没有等待路径的单位。

### Overflow 槽位

Overflow 半径按同一目标附近的全部活跃预留人口计算，而不是只看当前命令波次：

```text
spacing = largest radius * 2 + 4
core radius = spacing * (sqrt(population) * 0.62 + 1.5)
```

从外侧最多 6 圈确定性采样候选位置。候选必须满足：

- 位于世界内且不碰静态/动态障碍。
- 从当前点到候选点的静态路径可通行。
- 不与任何单位的预留槽位重叠。
- 不与任何单位当前占用重叠。
- 相同输入按 Unit ID、Group ID 和固定角度得到相同结果。

获得 Overflow 后，单位退出 V1 换槽集合，避免重新被交换回内部槽位。

## 3. 第二层：主动 Yielding

第二层不直接改变内部受阻单位的槽位，而是让其路径上的已就位单位暂时离开。每个让路者使用显式状态机：

```text
None
  → MovingAside（前往临时让路点）
  → Waiting（保持通道打开）
  → Returning（返回原槽位）
  → None
```

### 3.1 候选与触发

- 只处理 32 人以上编队和仍持有内部目标的 Moving 单位。
- 受阻单位连续停滞至少 120 Tick，或在目标区累计至少 240 Tick，才查找阻挡者。
- 目标区驻留超过 720 Tick 后不再启动新的 Yield；这时让现有让路结束，并留给 900 Tick Overflow 硬兜底。
- 阻挡者必须已经 Arrived，或距离自身槽位不超过 18px；不能正在让路、Overflow 或冷却。
- 只考虑靠近“受阻单位当前位置 → 原目标”线段的阻挡者，并用 Unit ID 解决相同代价的确定性排序。
- 每 30 Tick 最多启动 2 次，全局最多 4 个活动 Yield，避免终点附近同时大幅重排。

### 3.2 临时让路点

候选方向由路径切线的左右两侧及背离编队中心的方向组成，并用两档距离和固定权重确定性采样。每个候选必须：

- 在世界边界内，且单位圆盘和到达线段均不穿过静态/动态障碍。
- 不与任何单位当前占用或普通槽位预留冲突。
- 不与其他 Yield 的原槽位返回预留或临时让路点冲突。

开始让路后，原槽位不会被释放：`SlotTarget`、`DestinationYieldReturnTarget` 和 `DestinationYieldPoint` 同时参与预留检查。因此后续命令波次和 Overflow 都不能抢占让路者的原位置或临时停靠位置。

### 3.3 等待、返回与有界失败

- `MovingAside` 最多持续 360 Tick；到达让路点后进入 `Waiting`。
- 当受助单位到达/接近其槽位、命令已改变、进入 Overflow，或等待期限到达时，让路者开始返回。
- `Returning` 最多持续 180 Tick；成功后清理状态并进入 600 Tick 冷却，避免同一单位反复开合通道。
- 如果返回仍不能完成，已经验证安全的让路点会提升为该单位的永久 Overflow；状态机仍会结束，不产生新的永久卡死。
- 新 Move 或 Stop 会清理旧 Yield 上下文，防止旧协作关系污染新命令。

## 4. 与其他系统的边界

- 新 Move 和 Stop 会清空全部终点 V2 历史。
- 普通局部换槽保留总驻留历史。
- 动态路径重算和 `WaitingForPath` 不清空总驻留历史。
- Choke 控制期间不累计终点历史。
- Overflow 仍使用正常路径预算、Steering 和碰撞系统。
- Yielding 单位暂时退出局部换槽和 Overflow 候选集合，避免多个恢复器同时改写其目标。
- 后续 Move 会把 Overflow 槽位作为普通现有预留参与冲突检查。

## 5. 当前验收结果

### 临时编队阻挡

- 上一版：75/80 到达，5 个 Moving，最大误差 89.8px。
- 当前：80/80 到达，4 次 Yield，5 次 Overflow，结束时 0 个活动 Yield、0 Moving，最大误差 5.7px。

### 外圈封闭专用场景

测试先根据业务层可见的 `AssignedTarget` 选出 48 个最外层槽位，让它们先就位；随后释放 32 个内部槽位单位。

- 外圈：48/48 到达。
- 内部：32/32 到达。
- Yield：8。
- 结束时活动 Yield：0。
- Overflow：31；相比只有第一层时减少 1 次，表明已有内部单位穿过临时通道并回到原槽位体系。
- 最终 Moving：0。
- 重叠：0。
- 最大误差：7.2px。

### 其他改善

- 普通密集编队：80/80，最大误差 4.9px。
- 192 单位：192/192，最大误差 3.7px。
- 动态群组改道：48/48，最大误差 6.1px。

## 6. V2 后续层

Overflow 解决的是“不能永久卡住”，不等于最紧凑、最自然的最终编队。A2 仍保持当前优先级，后续顺序是：

1. 对 8～24 个局部单位重新匹配，支持三单位以上循环换槽，并把它放在 Yielding/Overflow 之前。
2. 为目标槽位计算 SlotDepth 和进入方向，让深层槽位优先进入而不是先形成外圈。
3. 为靠墙、角落、混合半径和动态建筑目标增加专用场景。
4. 调整终点专用 Steering，使已经打开的临时通道更容易被受助单位利用。
5. 继续把 Overflow 保持为所有有限协作恢复失败后的安全兜底。

Clearance 与 Movement Class 在 A2 完成后继续。
