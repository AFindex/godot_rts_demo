# SC2 风格地形系统专项计划

目标不是复制 SC2 编辑器界面，而是完成一套能够支撑同类玩法、适合 Godot、且不污染现有模拟内核的地形系统。

## 1. 最终边界

权威地形位于纯 C#：

- `TerrainMapSnapshot`：验证后不可变，可生成稳定字节和哈希。
- `ITerrainMapQuery`：寻路、建造、视野、战斗只依赖查询接口。
- `TerrainMapBuilder`：代码生成和编辑器导入共用的可变构建器。
- Godot Resource / 导入器：只负责把编辑数据转换成快照。
- `Rts3DTerrainPresenter`：只读快照并生成批量网格。

现有 `StaticWorld` 继续负责世界边界、静态矩形障碍和动态建筑；地形负责连续地表规则。未传入地形的旧场景保持平地行为。

## 2. 完整推进顺序

### T1：地形数据与第一条完整链路

实现并冻结：

- `0..15` 高度层。
- 独立材质、移动种类、可建造、坡道、遮挡标记。
- 地面 / 浮水 / 两栖 / 飞行通过规则。
- 圆形单位不能跨悬崖，能经坡道跨一层。
- 建筑只能放在同一平整且可建造区域。
- 3D 批量地表与崖壁。
- 单位、建筑、资源、弹道和鼠标拾取跟随地面高度。
- 稳定哈希、自测和一个程序生成测试地图。

验收不是“画面出现高低差”，而是同一份地形数据同时改变寻路、建造和显示；关闭地形后旧测试行为不变。

### T2：真正的高地视野与交战

实现：

- 视野源带观察高度。
- 低地到高地的遮挡传播。
- 坡道口附近的视野过渡。
- 高地获得目标视野后才允许攻击。
- 友军共享视野、探测、Burrow/Cloak 与地形视野共同工作。
- Smoke / Undergrowth 内外遮挡区域。

必须做真实 case：低打高无视野、空军提供视野、坡道探头、友军提供视野、目标退回高地丢失、弹道已发射后的处理。

### T3：地形数据工作流

实现 Godot 资源和编辑工具：

- 地形尺寸、格子大小和材质集。
- 刷高度层、表面、路径种类、可建造、菌毯限制、视野遮挡。
- 拉坡道时自动验证只连接相邻层。
- 实时显示 Ground / Water / Air / Build / Vision 覆盖层。
- 导出前验证孤立小岛、断裂坡道、窄于单位直径的假通路、资源或出生点不可达。
- 快照保存稳定版本和哈希；错误包含格子坐标和明确原因。

编辑器数据不能直接进入模拟。运行前必须转换为验证后的不可变快照。

### T4：与现有寻路的正式融合

实现：

- Clearance Bake 把地形哈希纳入来源哈希。
- Small / Medium / Large 三层静态烘焙读取悬崖和坡道净空。
- 高层路线从地形连通区域和坡道自动生成，不再要求全部手写 Portal。
- 坡道自动成为狭窄通道候选，并把真实宽度交给编队通行控制。
- 地形块修改时只重建受影响的寻路块。
- Dynamic Footprint、可破坏岩石、桥梁启停继续走动态占用，不重写地形本体。

这一步必须与底层寻路专项中的请求调度、邻居漏检分开：地形提供正确地图，不负责掩盖单位局部移动错误。

### T5：材质、装饰与地图可读性

实现：

- 多材质权重和边缘混合。
- 崖壁类型、坡道 trim、浅水边缘、不可建造板。
- Doodad 明确分为纯视觉、路径 Footprint、建造 Footprint、视野遮挡四类能力。
- 装饰批处理、距离裁剪、阴影级别和地图性能统计。
- 小地图地形颜色与高低层轮廓。

测试素材可以程序生成或使用项目自有素材，不依赖 SC2 原始贴图和模型。

## 3. 必须覆盖的真实测试

### 路径

- 同层直走。
- 隔着悬崖直点，必须绕到坡道。
- 大小单位过窄坡道。
- 单位圆边贴崖，不允许半身越界。
- 连续两层必须经过两个合法坡道。
- 地面、浮水、两栖、飞行在浅水和深水中的矩阵。
- 桥梁或岩石打开/关闭后连通性变化。

### 建造

- 同一平整可建造层成功。
- 跨悬崖边缘失败。
- 坡道失败。
- 浅水可走但建造失败。
- 视觉贴图变化不改变建造结果。
- Unbuildable Overlay 与建筑、单位占用共同给出稳定失败原因。

### 视野和战斗

- 低地无高地视野。
- 高地对低地。
- 空军或高地友军提供视野。
- Smoke / Undergrowth 内外矩阵。
- 探测不能绕过普通地形视野；共享视野可以传递已经获得的视野。

### 表现和输入

- 鼠标点击坡道、高地边缘和低地得到正确模拟坐标。
- 单位、建筑、资源、选择圈、弹道不悬空也不埋地。
- 地形材质更换不改变回放哈希。
- 远景地形节点和 Draw Call 不随格子数线性增加。

### T2 已完成（2026-07-15）

- `ITerrainMapQuery` 增加统一视线查询。地面观察者只能看见不高于“观察点地表高度 + 单位观察高度”的位置；高架/飞行观察者忽略悬崖和烟雾遮挡，但仍受自身视距限制。
- 坡道不使用四个点或固定触发距离。视线直接读取坡道上连续插值后的真实地表高度，因此侦察单位爬坡时会在自身观察高度足以越过上沿的实际位置获得高地视野。
- `BlocksVision` 区域形成明确的内外关系：外看内、内看外和射线穿过区域均遮挡；同一连续区域内部可互见。查询先以遮挡区域包围盒剔除，不让无关地图上的每条视线都扫描烟雾格。
- 单位配置新增观察高度和 Ground/Elevated 观察方式，并完整进入 Replay Package v30、Hot Snapshot v31、State Hash v32；非法高度和枚举在恢复时拒绝。
- 战斗选敌、指定目标持续合法性和建筑目标均消费玩家真实视野，不再把“没有隐形”错误等同于“全图可见”。探测只解决隐形目标，不能单独创造越过悬崖的普通视野；联盟共享的是已经计算完成的视野。
- 本项目明确冻结“合法发射的弹道在随后丢失视野时继续飞行并结算”的合同。它是本项目统一弹道规则，不冒充所有 SC2 武器的内部例外均已复刻。
- 新增五组生产模拟真实 case：低地拒绝攻击→高架观察→发射后丢失视野仍命中；连续坡道探头；高地盟友共享视野与探测组合；烟雾内外矩阵；目标退回高地后停止继续受击并解除交战。全部通过，专项汇总为 `hidden/reject/spot/launch/lostFlying/hit = True`。

### T3 已完成（2026-07-15）

- 新增可直接在 Inspector 编辑的 `RtsTerrainAuthoringResource`。尺寸、格子大小、悬崖高度、材质集、逐格数据、出生/资源/目标锚点和检查参数均为显式字段，不再把编辑数据藏在运行时 Base64 payload 中。
- 新增纯 C# `TerrainAuthoringDocument`。Height、Surface、Pathing、Buildable、Vision、Creep、Ramp/Clear Ramp 是互不代替的笔刷；圆形范围按格子中心计算，修改一个属性不会重写其他属性。
- 新增 `RtsTerrainAuthoring2D` 与 `addons/rts_terrain` 编辑器插件：2D 视图左键拖动绘制，一次笔画一次写回并进入 Godot UndoRedo；覆盖层可独立组合显示 Ground/Water/Air/Build/Vision/Creep/Ramp/Height/Surface/Validation。
- 地图改尺寸走显式重排：按旧列数和行数复制重叠格子，拒绝长度不匹配的数据，避免把旧一维数组按新宽度误读。
- 导出前检查断裂/越界坡道、孤立地面、出生点/资源点/目标点不可达，以及“格子中心连通但指定半径单位过不去”的窄缝。错误带稳定码和精确 `[column,row]`，检查失败不会输出运行时资产。
- 编辑数据只通过 `TerrainAuthoringResourceConverter` 转换为既有不可变 `TerrainMapSnapshot`；运行时资源继续使用版本化规范 payload 与稳定哈希。示例编辑资产为 `data/demo_terrain_authoring.tres`，示例输出为 `data/demo_authored_terrain.tres`。
