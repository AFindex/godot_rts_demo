# 战斗接触与推挤优先级

S8-E5a 把战斗中的碰撞响应从单纯 `Hold/Arrived/Idle/Moving` 扩展为有限接触角色。目标不是模拟真实质量，而是让已经建立的火力线和近战接触保持稳定，同时避免任何战斗单位成为永久不可推动的墙。

## 模块边界

`CombatContactPolicy` 是纯 C#、无状态策略。它只读取当前移动模式、战斗阶段、命令意图、近战/远程定位以及武器前摇/冷却移动能力，派生 `CombatContactSnapshot`。策略不写入 `CombatStore`、不增加 Profile 字段，也不进入 Replay Package、Hot Snapshot 或 State Hash。

`RtsSimulation` 在每次碰撞阶段开始前为存活单位计算一次接触快照。通用圆碰撞解算器只读取 `InverseMobility`，不知道武器、目标或攻击命令细节。因此后续调整角色规则不需要重写空间哈希、穿透检测或位置约束。

`PreviewCombatContact` 通过稳定门面公开角色、逆移动性、阻力等级和成对修正占比，诊断、测试与以后可能增加的 HUD 不需要读取内部数组。

2026-07-14 新增的 `UnitPushPriorityPolicy` 与本策略并列而不是替代它：Movement Class 决定普通行军的路权，Combat Contact 决定已经接敌后的站位阻力。活动目标存在时，同级单位不启用普通行军强推，避免追击者把固定前摇或近战槽位当成普通 Arrived 阻挡者；严格更高的 Movement Class 仍可按有限比例施加压力。

## 当前角色

| 角色 | 逆移动性 | 阻力等级 | 语义 |
|---|---:|---:|---|
| FixedWindup | 0.08 | 4 | 正在固定前摇，最不容易被挤离火力点，但仍可缓慢让位 |
| MeleeContact | 0.14 | 3 | 已经进入近战攻击状态，保持包围和接触面 |
| FixedCooldown | 0.30 | 2 | 固定武器已经开火，冷却阶段比前摇更容易让位 |
| MobileWeapon | 0.72 | 1 | 仅限 AttackMove 且当前武器阶段允许移动 |
| Standard | 0～1.00 | 0 | 沿用 Hold/Arrived/Idle/移动的普通碰撞规则 |

成对穿透修正按双方逆移动性归一化。面对逆移动性 0.65 的普通 Idle 单位时，四种战斗角色承担的修正占比分别为 `0.110 / 0.177 / 0.316 / 0.526`。双方都不可移动时维持原行为，不凭空选择随机一方。

移动武器必须同时处于 AttackMove，并在当前前摇或冷却阶段声明可移动。仅仅拥有移动射击能力、但执行 Hold 或显式 AttackTarget 时仍按固定武器处理，避免能力字段绕过命令语义。

## 验收与性能

`combat-contact-priority` 在四条隔离业务车道中覆盖固定前摇、近战接触、固定冷却、移动射击和作为挤压方的普通单位。随后给前三类施加完全相同的一 Tick 穿透压力，实测位移为 `0.32 / 0.52 / 0.93px`，与公开修正占比顺序一致。Package v17 / Hot v17 恢复最终 Hash 完全一致，证明接触角色可从权威状态重新派生。

当前格式为 Replay Package v29 / Hot Snapshot v30 / State Hash v31。接触角色仍然可以从权威战斗状态派生；动态阻塞窗口、推后 Reservation 迁移倒计时以及语义化移动目标与结果作为真实未来态保存。

显式攻击建筑的边缘恢复不属于碰撞阻力规则。它只在攻击者仍在真实射程外且当前追击已不可达或连续 90 Tick 没有进展时，读取当时存活友军占位重新选择完整矩形边缘；重新选择后继续使用本页定义的正常碰撞与接触阻力。`combat-attack-building` 实测 8/8 攻击者参与、只发生 2 次恢复、越界伤害 0。

通用推挤接入后的历史结果中，`combat-contact-priority` 的角色、阻力等级、修正占比和物理位移顺序保持不变。`combat-melee-slots` 因允许 8 个单位只有 7 个攻击仍通过、并把槽位就绪误当攻击资格，现已退出活动门禁；相关录像只作为旧行为审计证据。

E5a 到此收口。它不继续扩展成任意标签、阵营特判或脚本化质量矩阵。若实际大规模战线继续暴露问题，下一步应先增加一个有限的宏观战线疏通场景，再决定是否需要 E5b；没有失败场景时不继续调参。
