# War3 遭遇战导航调试器

这套工具只挂在 `War3Rts` 场景下，默认关闭，不修改寻路、避让或施工规则。地图加载后点击右上角常驻的 `GM 导航` 按钮打开，也可以按 `F7`；不要使用 `F8`，Godot 编辑器会把它解释为“停止运行项目”。也可使用以下启动参数：

```text
--war3-navigation-debug
--war3-navigation-debug-grid
```

第二个参数会同时打开面板和 Small 净空连通层。调试器开启期间会启用详细模拟计时，关闭后恢复原来的 profiling 状态。

## 面板操作

- 游戏内 `GM 导航` 按钮或 `F7`：显示或隐藏面板与世界叠加。
- `F9`：完整暂停/继续模拟；暂停期间 AI、压测刷新、模拟 Tick 和音频事件都不会前进。
- `F10`：暂停状态推进一个物理 Tick，适合定位状态切换瞬间。
- `清轨迹`：清空当前主选单位的历史移动轨迹。
- `导出 JSON`：把当前指标、待处理路径请求、施工状态、阻挡体、连通层、选中单位路径和状态变迁写入 `user://war3_navigation_debug_*.json`。

面板可分别打开运行时导航网格、全部单位路线、选中单位详细路径、静态阻挡和建筑真实占地。它还会按选中单位显示当前路径游标、高层路线、目标/槽位、实际与期望速度、碰撞修正、恢复阶段、停滞计数、让路状态、瓶颈队列及施工接近状态。主选单位为选中集合中 ID 最小的存活单位。

## 世界叠加

- 绿色：当前局部路径及单位导航半径。
- 紫色：高层路线与瓶颈定义。
- 黄色 / 青色：命令目标、最终槽位、目标范围和目标 Bounds。
- 浅绿 / 蓝色 / 红色：实际速度、期望速度、碰撞修正。
- 橙色 / 红色：停滞、恢复或不可达单位。
- 红色矩形：静态阻挡体。
- 橙色矩形：已进入导航网格的动态建筑阻挡体。
- 蓝色矩形：尚未硬提交的施工预留区；绿色十字和连线为施工接近点及其工人。
- 净空层红线：可走与不可走单元边界；紫线：不同连通分区边界。

## 排查农民到地基卡住

1. 选中建造农民，打开“当前路径”“目标/槽位”“阻挡体”和“移动轨迹”。
2. 在地基刚出现前按 `F9`，再用 `F10` 单步。
3. 对照面板里的 `construction` 状态、`foundation`、`access`、路径终点和 `nav-revision`。
4. 若蓝色预留区存在但没有橙色硬阻挡，问题发生在施工接近/清场阶段；若硬阻挡出现后路径跨越矩形，检查路径失效和重寻路；若路径正确但单位不动，检查碰撞修正、yield、stall 和 recovery。
5. 在异常帧点击“导出 JSON”，保存可复现的单位路径、施工快照和状态变迁。

净空连通层可能触发当前导航 revision 的增量快照刷新，因此只在主动勾选时生成，不参与普通 HUD 更新。

## 全图自动审计

专项模式会自动进入当前默认遭遇战地图，从基地附近生成一个真实农民，并恢复双方完整初始建筑占地。第一阶段对 Small 净空网格做全图连通采样；第二阶段对相同采样点逐个调用玩家移动命令、推进完整模拟 Tick、记录首次路径终态，然后 Stop 并把农民复位到基地。它不会加载表现模型，也不会修改普通遭遇战逻辑。

```text
--war3-navigation-audit
--war3-navigation-audit-stride=1
--war3-navigation-audit-ticks=2
--war3-navigation-audit-output=<directory>
```

`stride=1` 会覆盖 267×160 的全部 42,720 个导航格心。输出目录包含逐点 JSON、CSV、Markdown 分析、左右对照俯视 PNG 和可实时查看的 progress log；CI 可另外重定向 Godot 标准输出。发现底层/运行时不一致、非法路径、命令拒绝或超时时，进程返回非零结果，适合自动回归。
